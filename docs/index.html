Food Object Detection with YOLO-World

This project uses YOLO-World, an open-vocabulary object detection model, to detect food-related objects from images and a live webcam feed. The test dataset uses simple single-food images, such as an apple, banana, donut, pizza, and orange, so the model can focus on clear food objects instead of mixed bowls or cluttered meal scenes.

This explores how computer vision could support faster nutrition logging and food recognition.

Data

The dataset is controlled using a CSV image manifest. Each row contains an image ID, image source, and description. For this version, I selected images with one main food object at a time to improve detection clarity.

Feature Engineering / Image Preprocessing

Instead of using the images blindly, I converted YOLO's raw outputs into structured features: object label, confidence score, bounding box coordinates, and bounding box area. I also used a confidence threshold to filter weak predictions.

  • Food-only class list: the model uses labels from food_classes.txt.
  • Image selection: images are kept simple, usually one or two food objects at a time.
  • Image resizing: YOLO resizes images before inference.
  • Confidence thresholding: weak detections are filtered out using a confidence threshold.
  • Bounding boxes: predictions include x1, y1, x2, y2 coordinates.
  • Bounding box area: each detection is converted into a size feature.
  • Detection table: outputs are saved as reports/detections.csv.

Try Live Camera Detection

This project can also run on a webcam. Because this is a static portfolio page, the camera does not open directly inside the browser. To try the live demo, run the command below in Terminal.

Example of how the live camera detection view could look with food labels and bounding boxes.
python src/camera_food_detection.py

This opens your computer camera and detects food-related objects live using the labels from food_classes.txt. Press q in the camera window or use Control + C in Terminal to stop it.

Limitations

  • YOLO-World improves flexibility, but it can still confuse visually similar objects.
  • Lighting, blur, camera angle, and object size affect detection quality.
  • Food recognition is harder than general object detection because many foods look similar.
  • Mixed meal images can be difficult, so this project uses clearer single-food test images.
  • A future improvement would fine-tune YOLO on a custom food dataset.

How to Reproduce

1) Install

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

2) Run detection on images

python src/run_detection.py --manifest data/image_manifest.csv --model yolov8m-worldv2.pt --conf 0.55

3) Run live camera detection

python src/camera_food_detection.py

4) Preview portfolio page

python -m http.server 8000 --directory docs

Open http://localhost:8000/
